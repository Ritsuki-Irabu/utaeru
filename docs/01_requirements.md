# 要件定義書

**バージョン：** 1.0
**作成日：** 2026年6月
**ステータス：** MVP確定

---

## 1. プロジェクト概要

### 1.1 背景・目的

カラオケにおいて以下の課題が存在する。

- 久しぶりの曲で歌い出しが不安
- テンポやイントロが思い出せない
- 音を出して確認すると不自然
- イヤホン使用は場面によっては適さない

既存の音楽アプリは「音を再生する」前提であり、カラオケ室内での"無音確認"ニーズに対応していない。

本アプリはBPM振動により、音を出さずに身体感覚でテンポを再現し、歌い出しの不安を軽減することを目的とする。

### 1.2 ポジショニング

| 一般的な音楽アプリ | 本アプリ |
| --- | --- |
| 音再生前提 | 無音前提 |
| 練習用 | 本番直前用 |
| 採点志向 | 不安解消志向 |

---

## 2. ターゲット

### 2.1 メインターゲット

- 年齢：20〜35歳
- 利用シーン：飲み会・友人カラオケ
- 特性：歌うことは好きだが自信は不安定な層

### 2.2 利用シーン

- カラオケ室内
- 曲予約直前
- 自分の番が近づいたタイミング

---

## 3. ステークホルダー

| 役割 | 説明 |
| --- | --- |
| admin | 公開曲マスタの管理・タグ管理を行う管理者 |
| user | マイリストを管理し、BPM確認を行う一般ユーザー |

---

## 4. 機能要件

### 4.1 認証・ユーザー管理

| # | 要件 | 優先度 |
| --- | --- | --- |
| F-01 | ユーザーが新規登録できる | 必須 |
| F-02 | ユーザーがログイン・ログアウトできる | 必須 |
| F-03 | ユーザーにadmin / user のロールを付与できる | 必須 |

### 4.2 公開曲マスタ管理（admin）

| # | 要件 | 優先度 |
| --- | --- | --- |
| F-04 | adminが曲を登録できる | 必須 |
| F-05 | adminが曲を編集・削除できる | 必須 |
| F-06 | Spotify APIから曲のBPMを取得できる | 必須 |
| F-07 | 登録済み曲の一覧を取得できる | 必須 |

### 4.3 マイリスト管理（user）

| # | 要件 | 優先度 |
| --- | --- | --- |
| F-08 | userが公開曲マスタから自分のマイリストに曲を追加できる | 必須 |
| F-09 | userがマイリストの曲にメモを追加・編集できる | 必須 |
| F-10 | userがマイリストから曲を削除できる | 必須 |
| F-11 | userが自分のマイリスト一覧を取得できる | 必須 |
| F-12 | userがマイリストの曲にタグを付け外しできる | 必須 |
| F-13 | userが自分のマイリストをCSVでエクスポートできる | 必須 |

### 4.4 タグ管理

| # | 要件 | 優先度 |
| --- | --- | --- |
| F-14 | adminがタグを登録できる | 必須 |
| F-15 | タグ一覧を取得できる | 必須 |

---

## 5. 非機能要件

| # | 要件 | 内容 |
| --- | --- | --- |
| N-01 | 無音動作 | 音声再生機能を持たない |
| N-02 | 起動速度 | 起動3秒以内 |
| N-03 | バッテリー消費 | 低消費電力設計 |
| N-04 | セキュリティ | 他ユーザーのデータへのアクセス不可 |
| N-05 | 可用性 | API応答時間 1秒以内（通常時） |

---

## 6. 制約事項

- メロディ再現は行わない
- テンポ変化曲未対応（MVP時点）
- BPMデータ精度はSpotify API依存
- 拍子は4/4固定（MVP時点）

---

- 3拍子・8分音符対応
